# KI-gestütztes Test-Driven Development und Behaviour-Driven Development

Notizen zu meinem Vortrag auf den IT-Tagen 2025.

## Beispielprojekt
[ConferenceExample](https://github.com/andreaslausen/ConferenceExample)

## TDD-Prompts
```
Ich möchte folgende Änderung vornehmen: Ein Titel muss mindestens 3 Zeichen haben. Wir gehen nach TDD vor. Schreib zunächst nur einen Test.
```

```
Ich möchte einen neues Feature implementieren. Als Konferenzveranstalter möchte ich eingereichte Sessions akzeptieren können. Das Feature soll im Bounded Context *Conference* in einem ConferenceService implementiert werden. Wir gehen nach TDD vor. Schreib zunächst nur einen Unit Test.
```
## BDD-Prompts
```
Schreib einen Akzeptanztest für folgendes Feature: Als Speaker möchte ich einen Vortrag einreichen können, damit ich auf der Konferenz sprechen kann. Gehe nach BDD vor. Schreib zunächst nur das Featurefile (in englisch).
```

```
Ich möchte folgendes Feature hinzufügen: Als Speaker möchte ich einen eingereichten Beitrag bearbeiten können.
Gehe nach BDD vor. Schreib zunächst nur das Featurefile (in englisch).
```
